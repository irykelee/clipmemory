# 剪忆 ClipMemory v2.5.9 — 卡死检测 + 31 项审计修复 / Hang detection + 31 audit fixes

**发布日期 / Release Date**: 2026-07-21
**类型 / Type**: Minor release (observability + comprehensive audit)

## 中文

### 主要更新 (Highlights)

- **🛡 卡死检测（HangDetector）** — 主线程心跳 + 30s 探针；首次检测到主线程 60s 不响应即记录 stack 并自动恢复；避免 UI 真正卡死后无声无息
- **🛡 备份包 PBKDF2 升级** — 600k 轮 PBKDF2-SHA256 取代单轮 HKDF；弱口令离线暴力破解成本提升 ~10⁵ 倍（OWASP 2023 合规）；老包透明兼容
- **⚡ RTF 复制缓存桥接** — `copyToClipboard` RTF 分支命中 cache 后 < 1ms（之前每次重解析 20-100ms 阻塞主线程）；cache 跨 list/quickbar 自动桥接
- **🛡 UI 状态不丢失** — search bar 输入不再因 `@State didSet` 不经 Binding 触发导致键盘高亮残留；sidebar 标签 badge 不再因 tag 增减 stale
- **🛡 主线程 IO 后台化** — `copyToClipboard` image/RTF 路径不再阻塞剪贴板轮询；备份导出 50MB size guard 防 OOM

### 修复 (Fixes)

按影响排序 (high → medium → low)：

**High impact（数据 / 资源 / UX 关键路径）**

- **OCR 重试逻辑修复** — `markOCRAttempted` 移出 OCR 完成前调用；encrypt 失败不再永久标记，下次启动自动重试（BUG-010）
- **Keychain KeyStore 原子替换** — SecItemUpdate-first（OS-level atomic replace），errSecItemNotFound 时 fallback 到 SecItemAdd；消除 delete→add 窗口的潜在 key 丢失（L-3）
- **ImageStorage 迁移 OOM 防护** — `attributesOfItem` 先查 size 再 `Data(contentsOf:)`，避免超大 legacy 文件触发 OOM（BUG-028）
- **AppDelegate deinit 完整清理** — Carbon hotkey + clipboard monitor + welcome window 在 deinit 路径也清理；terminate 路径之前被释放不再泄漏（BUG-037）
- **NSCache 容量限制生效** — imageCache.setObject 传 `cost: data.count`，totalCostLimit (100MB) 真正触发驱逐；100 张大图不再超出 100MB 上限（BUG-027）
- **trimToMaxItems 保持时间序** — pinned items 不再被强行移到列表前面；8:00 pinned 不会显示在 9:00 non-pinned 之前（BUG-014）
- **PBKDF2 derivedKey 内存清零** — `SymmetricKey(data:)` 拷贝完成后立刻 `derivedKey.resetBytes(in:)`；本地内存取证场景下敏感 key 字节不留堆（NEW-2）

**Medium impact（线程安全 / 兼容性 / UX 一致性）**

- **feedURLString 跨线程读锁保护** — `OSAllocatedUnfairLock<String?>` 包裹 Sparkle delegate 读路径；修复主线程写 + 任意线程读的 data race（BUG-031）
- **imagesDirectory lazy var race fix** — 改为 `let` + 立即初始化闭包；saveImage 后台线程 + loadImageObject 主线程并发首次访问不再 double-init（BUG-026）
- **isPinned/isSensitive 向前兼容** — `decodeIfPresent ?? false` 替代 `decode(Bool)`；老持久化数据缺失字段不再 throw keyNotFound → 不再 wipe history（BUG-043）
- **UpdateStatus @MainActor** — 4 个 `@Published` 属性所在类标 @MainActor；编译器层面拦截跨线程 mutation（BUG-033）
- **searchText @State didSet → onChange** — TextField 经 Binding 写 `@State` 绕过 setter 的 SwiftUI gotcha；键盘高亮不再残留（BUG-004）
- **tagCounts cache 跟随 items 变化** — `.onChange(of: store.items)` 替代 `.items.count`；sidebar 标签 badge 在 tag 增删后立即更新（BUG-005）
- **AppPicker ForEach 用 bundleId** — `ForEach(filtered.indices, id: \.self)` 改 `\.bundleId`；app 启停时 AppPickerRow 的 @State (isHovered/resolvedIcon) 不再跳到错位（BUG-006）

**Low impact（清理 / 性能 / 边界正确性）**

- **loadKeyData 缓存回写** — miss 后回写 `cachedLoadedKey`；多次 BackupPackage export 不再重复查 Keychain（BUG-018）
- **跨字符串 String.Index 死代码删除** — `lt.distance(from: lt.startIndex, to: dsi)` 中 `dsi` 来自另一个 String 实例；Unicode 变长字符（"İ" → "i̇"）不再 fatalError 崩溃（BUG-008）
- **startOfDayBeforeYesterday -1 → -2** — 变量名语义对齐；unpinOlder "older than yesterday" 不再误判（BUG-016）
- **HotKey 默认 Cmd+Shift+V** — 对齐 AppDelegate:104 文档化意图与 macOS 剪贴板管理器惯例；只影响新装机用户（BUG-011）
- **HotKey EventHandler 失败路径清理** — `RegisterEventHotKey` 失败时 `RemoveEventHandler`；handler 不再在进程生命周期内残留（BUG-012）
- **bothDownKeepPrimary enum case** — 主+备都不可达时显式区分于 `automaticPrimaryDown`；UI 状态面板显示真实"两路都不可达"而非误导（BUG-035）
- **CJK 账户 token 检测** — `(?<![A-Za-z0-9])` 替换 `\b`；嵌入中文中的 API key / token 不再漏检（BUG-046）

### 升级提示 (Upgrade Note)

- v2.4.0 起带自动升级模块（Sparkle）的版本：等 App 内自动更新，或 `brew upgrade --cask clipmemory`
- 无数据迁移、无一次性弹窗
- **备份包兼容性**：v2.5.9 导出的 `.clipmemory` 包用 PBKDF2-SHA256，老版本可读；但反过来（老版本导出 → v2.5.9 导入）也已透明兼容，manifest 自带 `keyDerivationVersion` 字段指示算法

## English

### Highlights

- **🛡 Hang detection (HangDetector)** — Main-thread heartbeat + 30s probe; first hang after 60s of no response records stack and auto-recovers; prevents silent UI freezes
- **🛡 Backup PBKDF2 upgrade** — 600k-round PBKDF2-SHA256 replaces single-round HKDF; offline brute-force cost ~10⁵× higher (OWASP 2023 compliant); old packages transparently compatible
- **⚡ RTF copy cache bridge** — `copyToClipboard` RTF branch hits cache in < 1ms (previously 20-100ms sync parse blocking main thread); cache auto-bridges across list/quickbar
- **🛡 UI state preservation** — Search bar input no longer leaves stale keyboard highlight due to `@State didSet` bypassing via Binding; sidebar tag badges no longer stale on tag add/remove
- **🛡 Main-thread I/O offload** — `copyToClipboard` image/RTF paths no longer block clipboard poll; backup export 50MB size guard prevents OOM

### Fixes

Sorted by impact (high → medium → low):

**High impact (data / resource / UX critical paths)**

- **OCR retry logic fix** — `markOCRAttempted` moved out of pre-OCR call; encrypt failures no longer permanently mark item, next launch retries (BUG-010)
- **Keychain KeyStore atomic replace** — SecItemUpdate-first (OS-level atomic replace), SecItemAdd fallback on errSecItemNotFound; eliminates potential key-loss window in delete→add sequence (L-3)
- **ImageStorage migration OOM guard** — `attributesOfItem` checks size before `Data(contentsOf:)`, avoids allocating huge legacy files (BUG-028)
- **AppDelegate deinit complete cleanup** — Carbon hotkey + clipboard monitor + welcome window now cleaned in deinit path too; no leak if deallocated outside terminate path (BUG-037)
- **NSCache capacity limit effective** — imageCache.setObject passes `cost: data.count`; totalCostLimit (100MB) actually triggers eviction; 100 large images no longer exceed 100MB (BUG-027)
- **trimToMaxItems preserves time order** — pinned items no longer forced to top of list; 8:00 pinned item no longer appears before 9:00 non-pinned (BUG-014)
- **PBKDF2 derivedKey memory zeroing** — `derivedKey.resetBytes(in:)` immediately after `SymmetricKey(data:)` copy; sensitive key bytes don't linger on heap under local memory forensics (NEW-2)

**Medium impact (thread safety / compatibility / UX consistency)**

- **feedURLString cross-thread read lock** — `OSAllocatedUnfairLock<String?>` wraps Sparkle delegate read; fixes data race where main-thread writes + arbitrary-thread reads conflicted (BUG-031)
- **imagesDirectory lazy var race fix** — switched to `let` + immediate init closure; saveImage background thread + loadImageObject main thread concurrent first-access no longer double-init (BUG-026)
- **isPinned/isSensitive forward compat** — `decodeIfPresent ?? false` replaces `decode(Bool)`; old persisted data missing these fields no longer throws keyNotFound → no history wipe (BUG-043)
- **UpdateStatus @MainActor** — class annotated @MainActor; compiler-level enforcement of @Published cross-thread mutation prevention (BUG-033)
- **searchText @State didSet → onChange** — SwiftUI gotcha where TextField Binding writes bypass setter; keyboard highlight no longer stale (BUG-004)
- **tagCounts cache refresh on items change** — `.onChange(of: store.items)` replaces `.items.count`; sidebar tag badges update immediately on tag add/remove (BUG-005)
- **AppPicker ForEach uses bundleId** — `ForEach(filtered.indices, id: \.self)` → `\.bundleId`; AppPickerRow @State (isHovered/resolvedIcon) no longer jumps to wrong app on launch/quit (BUG-006)

**Low impact (cleanup / performance / boundary correctness)**

- **loadKeyData cache writeback** — cache miss writes back to `cachedLoadedKey`; multiple BackupPackage exports no longer re-query Keychain (BUG-018)
- **Cross-string String.Index dead code removed** — `lt.distance(from: lt.startIndex, to: dsi)` used `dsi` from different String instance; Unicode variable-width chars ("İ" → "i̇") no longer fatalError crash (BUG-008)
- **startOfDayBeforeYesterday -1 → -2** — variable name semantics aligned; unpinOlder "older than yesterday" no longer misclassifies (BUG-016)
- **HotKey default Cmd+Shift+V** — aligns with AppDelegate:104 documented intent + macOS clipboard-manager convention; affects only new installs (BUG-011)
- **HotKey EventHandler failure path cleanup** — `RemoveEventHandler` on `RegisterEventHotKey` failure; handler no longer lingering for process lifetime (BUG-012)
- **bothDownKeepPrimary enum case** — distinguishes both feeds down from primary-only down; UI status panel shows truthful "both feeds unreachable" instead of misleading "primary down" (BUG-035)
- **CJK account token detection** — `(?<![A-Za-z0-9])` replaces `\b`; API keys/tokens embedded in Chinese text no longer slip through detection (BUG-046)

### Upgrade Note

- v2.4.0+ with the auto-update module (Sparkle): wait for in-app auto-update, or run `brew upgrade --cask clipmemory`
- No data migration, no first-launch prompts
- **Backup package compatibility**: v2.5.9 exports use PBKDF2-SHA256; older versions can read them. Reverse direction (old export → v2.5.9 import) is also transparently compatible via `keyDerivationVersion` manifest field
