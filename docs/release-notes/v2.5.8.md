# 剪忆 ClipMemory v2.5.8 — 稳定性审计 + 23 项修复 / Stability audit + 23 fixes

**发布日期 / Release Date**: 2026-07-20
**类型 / Type**: Patch release

## 中文

### 主要更新 (Highlights)

- **🛡 备份导出 / 导入加固** — 卡住的 `ditto` 不再无限阻塞 UI（30s 超时 + SIGKILL 升级）；HKDF 盐用 OS CSPRNG 失败时显式报错，不再静默用零填充
- **⚡ RTF 解析移到后台队列** — 大体积富文本粘贴不再让剪贴板轮询卡顿；OCR/图片识别也走后台，主线程更顺
- **🛡 SwiftUI 渲染警告修复** — 列表项数变化触发的「Modifying state during view update」警告消除，无多余重复渲染
- **🔧 内存存储线程安全** — 测试与未来多线程 caller 不再因 `MemoryStorageBackend` 数组 mutation 崩溃 / 丢数据
- **🏷 标签色 fallback 修复** — 无效 hex 颜色回退到主题色，浅色 / 深色模式下都可见

### 修复 (Fixes)

- **CI 私钥落盘加固** — Sparkle EdDSA 私钥落盘前 umask 077 + chmod 600，删前 `rm -P` 三次覆写，避免短窗口泄露
- **CI Keychain 分区限缩** — 限缩到导入证书的 SHA-1（避免 macos-latest runner 上 `apple:` 通配的宽攻击面）
- **Script 脚本诊断捕获** — `Scripts/preflight.sh` 捕获 `xcodegen` 失败 stderr；`Scripts/test-ui.sh` 动态解析 DerivedData 路径（不再硬编码 hash）
- **ImageStorage XCTest 隔离** — 测试不跑 legacy 迁移、不动 `UserDefaults.standard`
- **ImageStorage Keychain 回退** — C-1 留下的 legacy 解码回退到 Keychain 加载 key，不再 `.decryptionFailed`
- **ClipboardMonitor delegate 化** — 7 处 `ClipboardStore.shared.X` 改为 delegate 访问（testable / 无单例依赖）
- **AppDelegate IUO → Optional** — 4 处隐式解包改 Optional，propagation 到 TipsView + ContentView
- **Tab/tag 计数缓存** — I-9 `@State` 缓存 + `.onChange(items.count)` 失效，sidebar badge 不再每帧重算

### 升级提示 (Upgrade Note)

- v2.4.0 起带自动升级模块（Sparkle）的版本：等 App 内自动更新，或 `brew upgrade --cask clipmemory`
- 无数据迁移、无一次性弹窗

## English

### Highlights

- **🛡 Backup export/import hardening** — Stuck `ditto` no longer blocks UI forever (30s timeout + SIGKILL escalation); HKDF salt now errors explicitly on CSPRNG failure instead of silently using zero-fill
- **⚡ RTF parse moved off the clipboard poll queue** — Large rich-text pastes no longer stall the 0.5s poll; OCR/image recognition also background
- **🛡 SwiftUI rendering warning fixed** — "Modifying state during view update" warnings on item-count changes eliminated, no more spurious extra renders
- **🔧 In-memory storage thread-safe** — Tests and future multi-thread callers no longer crash or lose data from `MemoryStorageBackend` array mutation
- **🏷 Tag color fallback fixed** — Invalid hex colors now fall back to accent color, visible in both light/dark mode

### Fixes

- **CI private-key on-disk hardening** — Sparkle EdDSA private key uses `umask 077` + `chmod 600` before write; `rm -P` triple-overwrite on delete to avoid short-window exfiltration
- **CI keychain partition narrowing** — Restricted to imported certificate's SHA-1 (avoid the `apple:` wildcard attack surface on `macos-latest` runners)
- **Scripts diagnostics capture** — `Scripts/preflight.sh` now captures `xcodegen` stderr; `Scripts/test-ui.sh` dynamically resolves DerivedData path (no more hardcoded hash)
- **ImageStorage XCTest isolation** — Tests no longer run legacy migration nor touch `UserDefaults.standard`
- **ImageStorage Keychain fallback** — Legacy decryption (post-C1) now falls back to Keychain-loaded key, no more `.decryptionFailed`
- **ClipboardMonitor delegate-ified** — 7 `ClipboardStore.shared.X` access sites converted to delegate calls (testable / no singleton dependency)
- **AppDelegate IUO → Optional** — 4 implicit unwraps converted to Optional, propagated to TipsView + ContentView
- **Tab/tag count caching** — I-9 `@State` cache + `.onChange(items.count)` invalidation; sidebar badge no longer recomputes every frame

### Upgrade Note

- v2.4.0+ with the auto-update module (Sparkle): wait for in-app auto-update, or run `brew upgrade --cask clipmemory`
- No data migration, no first-launch prompts

## 安装 / Install

```bash
# Homebrew Cask
brew tap irykelee/clipmemory
brew trust irykelee/clipmemory
brew install --cask clipmemory

# Manual download
# https://github.com/irykelee/clipmemory/releases/download/v2.5.8/ClipMemory.tar.gz
```

> **首次打开若提示「Apple 无法验证…」/ If macOS blocks the first launch with "Apple cannot verify…"**: 这是 macOS 对未公证应用的常规拦截，不是病毒。右键点 App →「打开」→ 再点「打开」；或 系统设置 → 隐私与安全性 →「仍要打开」。仅需操作一次。/ This is the standard prompt for non-notarized apps, not malware. Right-click the app → **Open** → **Open** again; or System Settings → Privacy & Security → **Open Anyway**. Only needed once.
