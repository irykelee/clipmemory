# 剪忆 ClipMemory v2.5.12 — 稳定性与数据安全大修 / Stability & data-safety overhaul

**发布日期 / Release Date**: 2026-07-25
**类型 / Type**: Patch release（60+ 项审计修复 + 发布管线加固 / 60+ audit fixes + release pipeline hardening）

## 中文

### 主要更新 (Highlights)

- **🛡 数据安全集中修复** — 全面代码审查后的 30+ 项修复：剪贴板历史不再因密钥初始化竞态整会话静默丢失（STOR-1）；更新源探测不再自我取消导致镜像容灾完全失效（UPD-1）；富文本条目恢复按内容搜索（CLIP-1）；图片条目支持去重，同一张截图重复复制不再产生重复文件和列表项
- **🖼 OCR 文字不再丢失** — 复制图片条目、导入备份、旧版图片迁移都不再清掉已识别的 OCR 文字（STOR-2）
- **⚡ 启动与操作更流畅** — 旧版图片迁移移出启动主线程；QuickBar 搜索结果缓存不再每次渲染重复过滤；标签面板打开只跑一遍分词管线；JSON 持久化编码移入后台队列
- **🔔 错误提示不再刷屏** — 加密失败弹窗按来源 60 秒聚合计数，OCR 回填失败时不再连环弹窗
- **💾 备份导入更安全** — 备份包解压校验符号链接与路径越界、JSON 读取加 100 MB 上限；`.incomplete` 标记删除失败不再静默吞错

### 修复 (Fixes)

**密钥与加密**

- **STOR-1 会话级静默丢历史（高危）** — `prepareKey()` 成功结果从不写入共享缓存，`getKey()` 首次 miss 后永久闩锁；登录自启动时钥匙串未解锁或全新安装的竞态下，整个会话的剪贴板历史与截图被静默丢弃。现在所有成功路径都发布到共享缓存
- **C-2 钥匙串锁定不再触发密钥重新生成** — 锁定返回 nil 等待下次会话，不再覆盖用户密钥
- **H-2 停止钥匙串连续重试** — 首次 miss 后闩锁，不再每次加密都访问 Keychain
- **H-3 加密失败通知带来源标签** — 为用户侧防抖弹窗提供依据（见 CLIP-3）
- **H-4 OCR 加密失败现在记录日志并通知**，不再静默跳过

**捕获与存储**

- **图片条目去重（CLIP-1）** — 图片也有 contentHash 指纹，同一张图重复复制只保留一条且不再落重复文件；命中时移到顶部并保留 OCR 字段
- **STOR-2 成员级重建丢字段（高危）** — 7 处逐字段重建 `ClipboardItem` 时漏传 `ocrText`/`ocrAttempted`，复制图片条目即永久清掉其 OCR 文字；新增 `with()` 复制 helper 根除这一类 bug
- **M-3 新装用户历史上限被误设为 1** — UserDefaults 缺省键经 clamp 变成 1，现在缺省回退 100
- **管线 CLIP-2 文本捕获 10 MB 上限** — 超限按多字节边界安全截断并记日志；JSON 持久化编码移入后台队列
- **STOR-3 旧版图片迁移移出启动主线程** — 大图库不再卡启动
- **STOR-4 图片迁移不再永不完成** — 永久不合格文件进 skip 列表，只有暂时性失败才重试
- **C-1 锁实现回退 NSLock** — 兼容 macOS 13（OSAllocatedUnfairLock 需要 14+）
- **C-3 imageStatus 读写串行化** — 关闭旧版迁移竞态
- **C-4 备份导入图片尺寸不可读时跳过**，不再中断整个导入
- **H-1 图片格式识别扩展** — 除 PNG/TIFF 外新增 JPEG/HEIC/GIF/BMP/JPEG-2000

**备份与导入**

- **BKP-2 备份包解压校验** — 拒绝符号链接与越出暂存目录的路径（防恶意包越界写）
- **BKP-3 备份 JSON 读取 100 MB 上限** — 超限按包损坏报错，不再无上限读入内存
- **BKP-1 `.incomplete` 标记删除失败按备份失败处理** — 好备份不再被误当残留剪掉；H-6 崩溃一致性剪枝标记配套

**界面与交互**

- **主界面 CLIP-1 富文本条目恢复可搜索（高危）** — 搜索与预览改用解密后的 RTF 明文缓存，不再对密文解析后回退为 "Rich Text"
- **次级 CLIP-1 QuickBar 与主窗口键盘监听冲突（高危）** — 按 Return 不再有两个 handler 各复制不同条目
- **主界面 CLIP-3 裁剪确认弹窗提升挂载层级** — 设置页调低历史上限时确认框能正常弹出
- **主界面 CLIP-4 QuickBar 过滤结果缓存** — 不再每次渲染重复 4+ 次 O(n) 过滤
- **次级 CLIP-2 新建标签按 Return 提交草稿**，不再直接关面板丢输入
- **次级 CLIP-3 标签建议只跑一遍管线** — 打开标签面板不再重复执行 regex+NLTagger
- **模型 CLIP-2 邮箱识别边界** — 句中、子域名、尾标点邮箱不再误判
- **架构 CLIP-2 复制 OCR 文本先标记自家写入**，不再被重新捕获成新条目
- **H-7/H-8 RTF 解析移出主线程**；H-9 合并重复 onChange；H-10 可见索引缓存；H-12 列表 store 引用修复；H-13 元组状态改 Equatable 结构；H-14 匹配计数缓存；H-19 行视图 Equatable 补齐字段

**基础设施与稳定性**

- **UPD-1 更新源探测不再自我取消（高危）** — `triggerProbe()` 取消了自身所在任务，镜像容灾在运行时完全失效；已修复且 fallback 决策正确记录
- **INFRA-1 热键注册失败路径内存泄漏** — CFRetain/PassRetained 平衡，重试不再覆盖未释放指针
- **INFRA-2 卡顿检测器按需捕获堆栈** — 不再每 5 秒无条件在主线程捕获（检测器自身造成卡顿）
- **H-15 Carbon 热键 passRetained 平衡**（防 use-after-free）；H-18 热键回调加锁防竞态
- **修复 CI 测试挂死** — 测试环境注入真实 app 时，加密失败弹模态框阻塞测试进程 60+ 分钟；现在测试环境跳过该弹窗

**更新与发布管线**

- **UPD-2 feed 1 MB 上限前置** — Content-Length 预检 + 流式超限即取消，不再整包入内存后才检查
- **H-20 feed 响应体 1 MB 上限**（防 OOM）
- **REL-1 发布并发组固定** — 连续 tag 不再竞态截断 appcast 提交
- **REL-2 Cask 自校验改为存在+语法检查** — 原校验在数学上不可能通过（本地 SDK ≠ CI SDK）
- **REL-3 appcast 公开推送移到 Release 创建之后** — 校验失败不再留下悬空公开引用
- **REL-4 appcast 插入幂等** — 重跑不产生重复条目；缺 `</channel>` 显式报错
- **REL-5 西语/葡语复数文案修复** — 顺序敏感的字符串替换改为显式翻译表（已发布的破碎文案一并修复）
- **REL-6 发布源分支检测 main 优先 + 多候选显式失败**，jsDelivr 缓存按实际分支刷新
- **CI 加固** — 两个 workflow 加超时上限；PR dry-run 修复签名（ad-hoc）；`gh release download` 位置参数语法；appcast 推送走所有者 PAT（admin bypass）解决分支保护拦截；Release 标题改用 `name:` 输入（双语标题不再变 stub）

### 升级提示 (Upgrade Note)

- v2.4.0 及以上：等 App 内自动更新，或 `brew upgrade --cask clipmemory`

## English

### Highlights

- **🛡 Data-safety overhaul** — 30+ fixes from a full code review: clipboard history can no longer be silently lost for an entire session due to a key-initialization race (STOR-1); update-feed probing no longer cancels itself, which had disabled the mirror fallback entirely (UPD-1); rich-text items are searchable by content again (CLIP-1); image entries are now deduplicated, so re-copying the same screenshot no longer creates duplicate files and list entries
- **🖼 OCR text no longer lost** — re-copying an image entry, importing a backup, or migrating legacy images no longer erases recognized OCR text (STOR-2)
- **⚡ Smoother startup and interaction** — legacy image migration moved off the launch main thread; QuickBar search results are cached instead of re-filtering on every render; the tag panel runs its tokenization pipeline only once; JSON persistence encoding moved to a background queue
- **🔔 No more alert storms** — encryption-failure alerts are coalesced per source over a 60-second window instead of stacking modal after modal
- **💾 Safer backup import** — package extraction validates symlinks and path traversal, JSON reads are capped at 100 MB, and `.incomplete` marker removal failures are no longer swallowed

### Fixes

**Keys & encryption**

- **STOR-1 Session-wide silent history loss (high)** — `prepareKey()` never published its success to the shared cache, so a first `getKey()` miss latched for the whole session; when the Keychain was still locked at login-start, or on a fresh-install race, every clipboard entry and screenshot of the session was silently dropped. All success paths now publish to the shared cache
- **C-2 Locked Keychain no longer triggers key regeneration** — returns nil and retries next session instead of overwriting the user's key
- **H-2 Stopped Keychain hammering** — latch after the first miss instead of querying on every encrypt
- **H-3 Encryption-failure notifications now carry a source tag** — enables the throttled user alert (see CLIP-3)
- **H-4 OCR encrypt failures are now logged and notified**, no longer silently skipped

**Capture & storage**

- **Image deduplication (CLIP-1)** — images now get a contentHash fingerprint, so re-copying the same image keeps a single entry and no duplicate file; a hit moves the entry to the top and preserves its OCR fields
- **STOR-2 Field loss in member-wise rebuilds (high)** — 7 rebuild sites dropped `ocrText`/`ocrAttempted` when reconstructing a `ClipboardItem`; re-copying an image entry permanently erased its OCR text. A new `with()` copy helper eliminates the whole bug class
- **M-3 Fresh installs no longer get a history cap of 1** — an absent UserDefaults key was clamped to the minimum; it now falls back to the 100 default
- **10 MB text capture cap (CLIP-2)** — oversized captures are truncated at a safe multibyte boundary and logged; JSON persistence encoding moved to a background queue
- **STOR-3 Legacy image migration moved off the launch main thread** — large libraries no longer stall startup
- **STOR-4 Image migration can actually complete** — permanently ineligible files go on a skip list; only transient failures are retried
- **C-1 Lock reverted to NSLock** — macOS 13 compatibility (OSAllocatedUnfairLock requires 14+)
- **C-3 imageStatus read/write serialized** — closes the legacy migration race
- **C-4 Backup image import skips files whose size is unreadable** instead of aborting the whole import
- **H-1 Broader image format detection** — JPEG/HEIC/GIF/BMP/JPEG-2000 in addition to PNG/TIFF

**Backup & import**

- **BKP-2 Extraction validation** — backup packages containing symlinks or paths escaping the staging directory are rejected (path-traversal hardening)
- **BKP-3 100 MB cap on backup JSON reads** — oversized blobs fail as corrupt instead of being read into memory unbounded
- **BKP-1 `.incomplete` marker removal failure now fails the backup** — good backups can no longer be pruned as residue (companion to the H-6 crash-consistency marker)

**UI & interaction**

- **Rich-text items searchable again (main CLIP-1, high)** — search and preview use the decrypted RTF plaintext cache instead of parsing ciphertext and falling back to a literal "Rich Text"
- **QuickBar vs main-window key listener conflict (secondary CLIP-1, high)** — pressing Return no longer races two handlers copying different entries
- **Trim-confirmation alert hoisted (main CLIP-3)** — the confirmation now appears when lowering the history cap from Settings
- **QuickBar filter results cached (main CLIP-4)** — no more 4+ O(n) filter passes per render
- **Return submits a new-tag draft (secondary CLIP-2)** instead of closing the sheet and losing the input
- **Tag suggestions computed once (secondary CLIP-3)** — opening the tag panel no longer runs the regex+NLTagger pipeline twice
- **Email detection boundaries (model CLIP-2)** — mid-sentence, subdomain, and trailing-punctuation addresses now classify correctly
- **Copying OCR text marks the write as our own (arch CLIP-2)** — it is no longer recaptured as a new entry
- **H-7/H-8** RTF parsing off the main thread; **H-9** duplicate onChange handlers merged; **H-10** visible-index caching; **H-12** list store reference fix; **H-13** tuple state replaced with an Equatable struct; **H-14** match-count caching; **H-19** row Equatable now covers all rendered fields

**Infrastructure & stability**

- **UPD-1 Update-feed probing no longer cancels itself (high)** — `triggerProbe()` cancelled the very task it ran on, which disabled the mirror fallback at runtime; fixed, and fallback decisions are recorded correctly
- **INFRA-1 Hotkey registration failure-path leak** — CFRetain/PassRetained balanced; retries no longer overwrite an unreleased pointer
- **INFRA-2 Hang detector captures stacks only on suspected hangs** — no longer unconditionally every 5 s on the main thread (the detector itself caused jank)
- **H-15** Carbon hotkey passRetained balanced (use-after-free); **H-18** hotkey callback lock against the Carbon/main race
- **CI test hang fixed** — tests inject into the real app, and a modal encryption-failure alert blocked the test process for 60+ minutes; the alert is now skipped under XCTest

**Update & release pipeline**

- **UPD-2 Feed 1 MB cap enforced early** — Content-Length pre-check plus streaming cancellation, instead of checking after the whole body is in memory
- **H-20 Feed response body capped at 1 MB** (OOM prevention)
- **REL-1 Fixed release concurrency group** — back-to-back tags no longer race-truncate the appcast commit
- **REL-2 Cask self-check reduced to existence + syntax** — the old sha256/version equality gate was mathematically unsatisfiable (local SDK ≠ CI SDK)
- **REL-3 Appcast is now published after the GitHub Release exists** — a failing pre-release verify no longer leaves a dangling public reference
- **REL-4 Appcast insertion is idempotent** — re-runs no longer duplicate items; a missing `</channel>` fails loudly
- **REL-5 Spanish/Portuguese plural wording repaired** — order-dependent string rewriting replaced with an explicit translation table (broken strings already shipped are fixed too)
- **REL-6 Release source-branch detection prefers main and fails loudly on ambiguity**; jsDelivr cache purged for the actual branch
- **CI hardening** — job timeouts on both workflows; PR dry-run ad-hoc signing; `gh release download` positional tag; appcast push via owner PAT (admin bypass) to clear branch protection; release title passed via the `name:` input (bilingual titles no longer fall back to the stub)
