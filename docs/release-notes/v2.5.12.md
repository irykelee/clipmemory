# 剪忆 ClipMemory v2.5.12 — 稳定性与数据安全大修 / Stability & data-safety overhaul

**发布日期 / Release Date**: 2026-07-25
**类型 / Type**: Patch release（60+ 项审计修复 + 发布管线加固 / 60+ audit fixes + release pipeline hardening）

## 中文

### 主要更新 (Highlights)

- **🛡 数据安全集中修复** — 全面代码审查后的 30+ 项修复：剪贴板历史不再因密钥初始化竞态整会话静默丢失（STOR-1）；更新源探测不再自我取消导致镜像容灾完全失效（UPD-1）；富文本条目恢复按内容搜索（CLIP-1）；图片条目支持去重，同一张截图重复复制不再产生重复文件和列表项
- **🖼 OCR 文字不再丢失** — 复制图片条目、导入备份、旧版图片迁移都不再清掉已识别的 OCR 文字（STOR-2）
- **⚡ 启动与操作更流畅** — 旧版图片迁移移出启动主线程；QuickBar 搜索结果缓存不再每次渲染重复过滤；标签面板打开只跑一遍分词管线；JSON 持久化编码移入后台队列
- **🔔 错误提示不再刷屏** — 加密失败弹窗按来源 60 秒聚合计数，OCR 回填失败时不再连环弹窗
- **💾 备份导入更安全** — 备份包解压校验符号链接与路径越界、JSON 读取加 100 MB 上限；`.incomplete` 标记删除失败不再静默吞错

### 修复 (Fixes)

- **捕获与存储**：文本捕获 10 MB 上限（多字节边界安全截断）；新装用户历史上限不再被误设为 1（M-3）；图片迁移区分永久不合格与暂时失败，不再永不完成（STOR-4）
- **界面**：裁剪确认弹窗在设置页也可触达；新建标签按 Return 不再丢草稿；QuickBar 与主窗口快捷键监听冲突修复；富文本条目可搜索；OCR 文本复制不再被重新捕获为新条目
- **基础设施**：热键注册失败路径内存泄漏；卡顿检测器不再每 5 秒无条件捕获堆栈；修复 CI 测试挂死（测试环境弹模态框）
- **发布管线**：CI 增加超时保护；appcast 推送修复分支保护拦截；西语/葡语复数文案修复

### 升级提示 (Upgrade Note)

- v2.4.0 及以上：等 App 内自动更新，或 `brew upgrade --cask clipmemory`

## English

### Highlights

- **🛡 Data-safety overhaul** — 30+ fixes from a full code review: clipboard history can no longer be silently lost for an entire session due to a key-initialization race (STOR-1); update-feed probing no longer cancels itself, which had disabled the mirror fallback entirely (UPD-1); rich-text items are searchable by content again (CLIP-1); image entries are now deduplicated, so re-copying the same screenshot no longer creates duplicate files and list entries
- **🖼 OCR text no longer lost** — re-copying an image entry, importing a backup, or migrating legacy images no longer erases recognized OCR text (STOR-2)
- **⚡ Smoother startup and interaction** — legacy image migration moved off the launch main thread; QuickBar search results are cached instead of re-filtering on every render; the tag panel runs its tokenization pipeline only once; JSON persistence encoding moved to a background queue
- **🔔 No more alert storms** — encryption-failure alerts are coalesced per source over a 60-second window instead of stacking modal after modal
- **💾 Safer backup import** — package extraction validates symlinks and path traversal, JSON reads are capped at 100 MB, and `.incomplete` marker removal failures are no longer swallowed

### Fixes

- **Capture & storage**: 10 MB text capture cap (multibyte-safe truncation); fresh installs no longer get their history cap mis-set to 1 (M-3); image migration distinguishes permanently ineligible files from transient failures and can actually complete (STOR-4)
- **UI**: trim-confirmation alert reachable from Settings; Return no longer discards a new-tag draft; QuickBar/main-window key listener conflict fixed; rich-text items searchable; copied OCR text is no longer recaptured as a new entry
- **Infrastructure**: hotkey registration failure-path memory leak; hang detector no longer captures stacks unconditionally every 5 s; CI test hang fixed (modal alert in the test host)
- **Release pipeline**: CI timeouts; appcast push unblocked from branch protection; Spanish/Portuguese plural wording repaired

### Upgrade Note

- v2.4.0+: wait for in-app auto-update, or run `brew upgrade --cask clipmemory`

brew trust irykelee/clipmemory
brew install --cask clipmemory

# Manual download
