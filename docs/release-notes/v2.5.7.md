# 剪忆 ClipMemory v2.5.7 — HangDetector observability module + key bug fixes / HangDetector 观测模块 + 关键 bugfix

**发布日期 / Release Date**: 2026-07-20
**类型 / Type**: Patch release

## 中文

### 主要更新 (Highlights)

- **🛰️ HangDetector observability module** — 后台 watchdog 自动检测主线程卡死 >60s 并记录完整堆栈 + 恢复时间，方便事后定位 2026-07-19 那类「sidebar OK 主区空」的疑难 bug。启动后自动运行，无任何用户感知。

- **🛡️ Fix silent data loss when HMAC fails** — 极少数情况下（Keychain 异常）复制新内容会被当重复项丢弃。本次修复后真实数据不会丢失。

- **🛡️ Fix QuickBar keyboard navigation crash** — 选中项被外部删除后按 ↑↓ 不再 OOB crash，单步选下一项会自动 clamp 到合法范围。

### 修复 (Fixes)

- **Test force-unwrap crash** — 2 个单元测试的 XCTAssertNotNil + `!` 模式改成 `guard let ... XCTFail(...) return`，测试失败时优雅记录而非 SIGILL 测试进程。
- **Image load concurrency race** — 启动期间 legacy 图片迁移多线程并发，async 写串行化避免数据竞争。
- **Excluded-app config TOCTOU** — `excludedBundleIds` 增原子 compound API（`updateExcludedBundleIds { ... }`），并发 add/remove 不再静默丢失条目。
- **Bulk-select toolbar stale state** — 主窗口按单行删除按钮后，批量选择工具栏正确消失（之前因 `selectedItems` 状态不同步而残留）。

### 升级提示 (Upgrade Note)

- v2.5.6 及以上：等 App 内自动更新，或 `brew upgrade --cask clipmemory`
- 无数据迁移、无一次性弹窗

## English

### Highlights

- **🛰️ HangDetector observability module** — Background watchdog automatically detects main-thread hangs >60s and records the full call stack + recovery time. Per-Apple-instrumented, no UI change. Helpful for post-mortem on hard-to-reproduce bugs like the 2026-07-19 "sidebar OK but main empty" issue.

- **🛡️ Fix silent data loss when HMAC fails** — On rare Keychain-access errors, new clipboard content was being treated as a duplicate and silently dropped. Now retains real data even under HMAC failure.

- **🛡️ Fix QuickBar keyboard navigation crash** — When the currently-selected item was deleted externally (e.g. full-window side), pressing ↑↓ no longer traps on out-of-bounds subscript. Bounds-clamped for safe fallback.

### Fixes

- **Test force-unwrap crash** — Two unit tests replaced the soft `XCTAssertNotNil` + `!` pattern with `guard let ... XCTFail(...) return`. Test failures now record gracefully instead of aborting the test process via SIGILL.
- **Image load concurrency race** — Multi-threaded legacy image migration during startup now serializes through a dedicated `DispatchQueue`, eliminating race on concurrent PNG re-encrypt writes.
- **Excluded-app config TOCTOU** — Added atomic `updateExcludedBundleIds { ... }` API to `ClipboardMonitor`. Concurrent add/remove of excluded bundle IDs no longer silently drops entries.
- **Bulk-select toolbar stale state** — Main window's bulk-select toolbar now correctly dismisses after a per-row delete. Previous behavior kept the toolbar visible with a stale UUID after the underlying item disappeared.

### Upgrade Note

- v2.5.6+: wait for in-app auto-update, or run `brew upgrade --cask clipmemory`
- No data migration, no first-launch prompts
