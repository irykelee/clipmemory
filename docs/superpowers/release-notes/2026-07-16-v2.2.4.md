# ClipMemory v2.2.4

**发布日期**：2026-07-16
**类型**：发布卫生修复（Release Hygiene）

## 修复

v2.2.4 不引入新功能；本次只修复 v2.2.3 发布路径上残留的发布卫生问题，让版本号、Cask、打包脚本、UI 标签和文档五处保持一致。

- **F-1 — 版本号与发布标签同步**：把 `project.yml` 的 `MARKETING_VERSION` 与 `CURRENT_PROJECT_VERSION` 升级到 `2.2.4`，并重新生成 `ClipMemory.xcodeproj/project.pbxproj` 中四个区段的版本号（Release / Debug × `MARKETING_VERSION` / `CURRENT_PROJECT_VERSION`）。修复 v2.2.3 切标签但未同步版本号，导致下游 Cask 与 Homebrew tap 拿到旧版本号的应用包。
- **F-2 — Homebrew Cask 版本与 SHA256 同步**：把 `Casks/clipmemory.rb` 的 `version` 改为 `"2.2.4"`，`sha256` 更新为 `Releases/ClipMemory.tar.gz` 的实际 SHA256。v2.2.3 的 Cask SHA 因打包脚本嵌套 bug 而与产物不一致，本次一并修正。
- **F-3 — `Scripts/package.sh` 默认版本号防呆**：把 `VERSION=${1:-2.0.0}` 改为默认从 `project.yml` 读取 `MARKETING_VERSION`，并在读取失败时显式报错退出，避免在不带参数调用时静默打包一个旧版本号的 tarball。
- **F-4 — 文档版本号同步**：8 份 README（根 `README.md` + 7 份 `docs/lang/README_*.md`）的顶部标题与更新日志行统一改为 `v2.2.4`，`docs/roadmap.md` 的版本与日期同步更新。
- **F-5 — v2.2.3 发布审计记录**：v2.2.3 的 release commit（`a57cdff`）标题宣称含「version」修复，但 `MARKETING_VERSION` 并未实际升级。本次同步将审计结论写入文档，避免再次出现「切标签但漏改版本」的问题复现。
- **F-6 — SwiftLint `--quiet` 清零**：清理两处 `--quiet` 仍报出的告警（`ContentView.swift` 的孤立 `swiftlint:enable identifier_name`、`ClipboardItemTests.swift` 与 `ContentView.swift` 的两处命名/长度告警）。`swiftlint lint --quiet` 现在以退出码 0 通过。
- **S-3 — Quick Bar 「打开完整窗口」快捷键标签修正**：移除 `ClipMemory/Views/QuickBarView.swift:137` 上误导性的 `⌘⌃V` 标签。实际行为是 `Cmd+Ctrl+V` 打开主窗口、菜单栏 📋 图标左键打开 Quick Bar，本次仅删除错误标签，不改绑定。
- **S-3 docs — 8 份 README 全局快捷键描述重写**：根 `README.md` 与 7 份语言 README 中关于 `Cmd+Ctrl+V` 的行重写，明确该快捷键打开主窗口而非 Quick Bar。
- **打包脚本产物展平**：把 `Scripts/package.sh` 中 `cp -r ... /tmp/ClipMemory.app` 替换为 `rm / mkdir / cp -r contents`，修复之前 `ClipMemory.app` 被嵌套在 `/tmp/ClipMemory.app` 内的 bug，使 `Releases/ClipMemory.tar.gz` 解包后直接得到一个可用的 `ClipMemory.app/`。

## 发布与兼容性

- **`v2.2.3` 完全未触碰**：v2.2.3 tag 仍指向其原始 commit `d27040f547fd907c3dbeddc89a81f53f556fd2ec`，v2.2.4 不修改 v2.2.3 的 tag、Cask、release notes 或 GitHub Release。
